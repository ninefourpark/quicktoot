# QuickToot

Mastodon 快捷发嘟工具 ・ 想嘟就嘟

QuickToot 是一款 为 Mastodon 用户设计的 浏览器插件，解决「只想发个嘟，结果刷了半小时时间线」的痛点。🐘

- 键盘召唤： 无论你在看什么网页，按一下快捷键就直接弹出输入框，写完即走。
- 丝滑串文： 同话题的嘟文可自动回复给上一则，形成嘟文串。
- 预填模板： 支持预填常用标签和文字（如：#读书记录、Day X），助力懒人发嘟。
- 隐私保障： 你的数据只存在本地浏览器。你可以随时将所有数据导出为 JSON 文件进行备份。

## 在开始尝试之前，你可能想知道...
-   **支持哪些站点？**
    只要是使用标准 Mastodon API 的站点都支持。
    
-   **支持哪些浏览器？**
    大多数 Chromium 内核的浏览器皆可使用。
    
-   **QuickToot 的作者能看到我的模板或话题吗？**
    不能。所有数据仅保存在你的浏览器中。 数据传输仅在你的浏览器与 Mastodon 实例之间进行。插件作者无法看到你的内容，包括你的模板和话题。
  
-   **会收费吗？**
    免费且开源。

## 安装指南

1. 前往 [下载页面](https://github.com/ninefourpark/QuickToot/releases) 获取最新版本（点击“Source code zip”），下载后解压到你的电脑中。
2. 打开浏览器，进入“扩展程序”页面（也可以直接在地址栏输入 chrome://extensions/）。
3. 开启右上角的“开发者模式”。
4. 点击左上角的“加载已解压的扩展程序”，选择刚才解压的文件夹。——大功告成！🎉

你可以查看 [更新日志](https://ninefourpark.github.io/projects/QuickToot/changlog.html) 了解新功能，或订阅 RSS，第一时间获得更新通知。


## 键盘操作一览

`Ctrl + Shift + Q` 
打开 QuickToot 插件面板。面板出现后，焦点会自动落在话题清单的第一条。

`↑ / ↓` 
在话题清单中上下移动选择话题。

`Enter` 
当话题获得焦点时，按 Enter 可触发嘟文撰写窗口。

`Tab` 
在面板内切换焦点，用于键盘操作整个界面。

`Ctrl/Cmd + Shift + 1/2/3` 
前 3 个话题默认设置专用快捷键。不需要打开插件面板，直接触发对应话题的撰写窗口。
快捷键可自定义。点击话题右侧的“更多”按钮（...）→“快捷键”进行设置。可指定打开撰写界面或网页链接。


## 常见问题

1.  **如何启用串文功能？ / 串文功能是如何运作的？**
    
    在设置中勾选「启用串文」，并授权 Mastodon。QuickToot 会开始记录你在各话题下发出的最近一条嘟文的 ID。当你在同话题下再次发嘟时，系统默认将新嘟文以“回复”形式接续在旧嘟文之后，自动形成串文。
    
2.  **有时想回复串文，有时不想，可以选择吗？**
    
    可以。即使在设置中勾选了「启用串文」，发布前你仍可在编辑器中自由切换“回复已有串”或“作为新串第一条嘟文”。
    
3.  **我已有一条串文了，QuickToot 发的嘟嘟可以回复到之前建立的已有串吗？**
    
    可以，QuickToot 支持绑定已有串。在话题旁点击“更多”(...) →“绑定串文”，并输入你想回复的那条嘟文 ID 。QuickToot 就会从该处开始自动串文。
    
4.  **为什么 @ 提及不生效？**
    
    QuickToot 不支持 @ 提及功能。QuickToot 专为个人日志、心情吐槽与日常记录设计。
    
5.  **不想用嘟文模板，可以关闭吗？**
    
    可以，在设置中清空「嘟文模板（全局）」并保存。
    
6.  **为什么我回复串文时不能选“公开”？**
    
    回复会继承原串的可见性，不能高于原串设置。
    
7.  **清除浏览器缓存会删除数据吗？**
    
    是的。数据皆存在本地。若你清除浏览器数据，QuickToot 的设置也会随之抹除。 **小贴士：建议定期导出备份。**
